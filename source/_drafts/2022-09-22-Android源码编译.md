# 应用无法使用@hide方法

应用开发的时候会依赖Android SDK，出于安全考虑，SDK中对部分类和方法标记了@hide，应用无法使用这些方法，编译会报错。

解决方法：

1. 将应用和源码一起编译。（原生应用一般是这么做的，不用AS不是很方便）
2. 创建适配器，使用反射调用hide方法（方法多的时候较复杂）
3. 创建同名类和方法，空实现，保证编译通过。运行时会使用BootClassLoader加载的类（方法多的时候较复杂）
4. 自行编译framework
   1. 获取到Android系统源码，编译出`framework.jar`：一般在`alps\out\target\common\obj\JAVA_LIBRARIES\framework_intermediates\classes.jar`下
   2. 使用compileOnly依赖
   3. 由于AS会优先使用Android SDK，因此需要修改下编译配置

```groovy
// app/build.gradle
//方法1：Android Studio 4.2之前
gradle.projectsEvaluated {
    tasks.withType(JavaCompile) {
        options.compilerArgs.add('-Xbootclasspath/p:libs/framework.jar')
    }
}
//方法2：Android Studio 4.2之后
gradle.projectsEvaluated {
    tasks.withType(JavaCompile) {
        Set<File> fileSet = options.bootstrapClasspath.getFiles()
        List<File> newList = new ArrayList<>();
        newList.add(new File("./libs/framework.jar"))
        newList.addAll(fileSet)
        options.bootstrapClasspath = files(newList.toArray())
    }
}
```
