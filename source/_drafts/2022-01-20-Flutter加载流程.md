# Android启动流程

以Flutter工程方式为例，模块集成方式类似。只摘取关键代码。

这种方式下MainActivity继承FlutterActivity。主要作用如下：

1. 选择Dart代码执行路径和入口
2. 选择Flutter初始路由
3. Host接口提供子类Hook，可以配置FlutterEngine。

查看FlutterActivity源码：绑定代理类，通过代理类创建FlutterView

```java
//./shell/platform/android/io/flutter/embedding/android/FlutterActivity.java
public class FlutterActivity extends Activity 
  implements FlutterActivityAndFragmentDelegate.Host, LifecycleOwner {
  @Override
  protected void onCreate(@Nullable Bundle savedInstanceState) {
    // 通过代理的方式，使FlutterActivity和FlutterFragment共用代理类逻辑，避免重复编码
    // Activity和Fragment继承Host接口，可以实现不同Hook方法逻辑
    delegate = new FlutterActivityAndFragmentDelegate(this);
    delegate.onAttach(this);
    delegate.onRestoreInstanceState(savedInstanceState);

    // 通过代理类创建FlutterView，添加到界面上
    setContentView(createFlutterView());
  }
  @NonNull
  private View createFlutterView() {
    return delegate.onCreateView(
        /* inflater=*/ null,
        /* container=*/ null,
        /* savedInstanceState=*/ null,
        /*flutterViewId=*/ FLUTTER_VIEW_ID,
        /*shouldDelayFirstAndroidViewDraw=*/ getRenderMode() == RenderMode.surface);
  }
}
```

查看代理类源码：

1. 创建Flutter引擎
2. 配置**PlatformPlugin（平台插件**）：用于接收Flutter消息，调用系统方法，例如播放声音，状态栏、导航栏、剪贴板等
3. 配置引擎：例如注册**FlutterPlugin（Flutter插件）**
4. 创建FlutterSurfaceView或者FlutterTextureView：默认使用SurfaceView，无法像普通View一样使用动画，或者使用z-index
5. 创建FlutterView，作为FlutterSurfaceView的父View，并添加首帧渲染监听，最终会注册到FlutterJNI中，由Native层调用
6. FlutterView绑定引擎
7. 如果配置了闪屏页，还会启动闪屏页，在Flutter引擎初始化、到首帧渲染完成之间显示。

```java
// ./shell/platform/android/io/flutter/embedding/android/FlutterActivityAndFragmentDelegate.java
class FlutterActivityAndFragmentDelegate {
  FlutterActivityAndFragmentDelegate(@NonNull Host host) {
    this.host = host; // 保存FlutterActivity对象
  }
  void onAttach(@NonNull Context context) {
    // 创建Flutter引擎
    setupFlutterEngine();
    // 创建PlatformPlugin，和Activity绑定
    // Fragment每次绑定Activity的时候都会重新创建
    platformPlugin = host.providePlatformPlugin(host.getActivity(), flutterEngine);
    // 配置Flutter引擎
    host.configureFlutterEngine(flutterEngine);
  }
  @NonNull
  View onCreateView(LayoutInflater inflater, @Nullable ViewGroup container, @Nullable Bundle savedInstanceState) {
    // 创建FlutterSurfaceView，用于提供绘制Flutter UI的Surface。也可以是FlutterTextureView
    FlutterSurfaceView flutterSurfaceView = new FlutterSurfaceView(host.getContext(), host.getTransparencyMode() == TransparencyMode.transparent);
    // 通知子类Hook
    host.onFlutterSurfaceViewCreated(flutterSurfaceView);
    // 创建FlutterView
    flutterView = new FlutterView(host.getContext(), flutterSurfaceView);
    // 添加首帧渲染监听，通知子类Hook，最终会注册到FlutterJNI中，由Native层调用
    flutterView.addOnFirstFrameRenderedListener(flutterUiDisplayListener);
    // 绑定Flutter引擎
    flutterView.attachToFlutterEngine(flutterEngine);
    //显示闪屏页，从Manifest中读取
    SplashScreen splashScreen = host.provideSplashScreen();
    if (splashScreen != null) {
      FlutterSplashView flutterSplashView = new FlutterSplashView(host.getContext());
      flutterSplashView.setId(ViewUtils.generateViewId(FLUTTER_SPLASH_VIEW_FALLBACK_ID));
      flutterSplashView.displayFlutterViewWithSplash(flutterView, splashScreen);
      return flutterSplashView;
    }
    return flutterView;
  }
  ...
  return flutterSplashView;
}
```

首先看下`setupFlutterEngine()`方法如何创建FlutterEngine：

1. 从`FlutterEngineCache`缓存中获取预热的引擎，重写`getCachedEngineId()`
2. 子类自行创建引擎，重写`provideFlutterEngine`方法
3. 默认创建引擎

```java
class FlutterActivityAndFragmentDelegate { 
  void setupFlutterEngine() {
    // First, check if the host wants to use a cached FlutterEngine.
    String cachedEngineId = host.getCachedEngineId();
    if (cachedEngineId != null) {
      flutterEngine = FlutterEngineCache.getInstance().get(cachedEngineId);
      isFlutterEngineFromHost = true;
      return;
    }

    // Second, defer to subclasses for a custom FlutterEngine.
    flutterEngine = host.provideFlutterEngine(host.getContext());
    if (flutterEngine != null) {
      isFlutterEngineFromHost = true;
      return;
    }
   
    flutterEngine = new FlutterEngine(host.getContext(), host.getFlutterShellArgs().toArray(),
            /*automaticallyRegisterPlugins=*/ false,
            /*willProvideRestorationData=*/ host.shouldRestoreAndSaveState());
    isFlutterEngineFromHost = false;
  }
}
```

继续看`attachToFlutterEngine`做了什么事情

```java
public void attachToFlutterEngine(@NonNull FlutterEngine flutterEngine) {
    this.flutterEngine = flutterEngine;

    FlutterRenderer flutterRenderer = this.flutterEngine.getRenderer();
    renderSurface.attachToRenderer(flutterRenderer);

    // 将安卓系统配置发给Flutter，例如字体缩放、日期格式、夜间模式、Locale等
    sendUserSettingsToFlutter();
    localizationPlugin.sendLocalesToFlutter(getResources().getConfiguration());

    flutterEngine.getPlatformViewsController().attachToView(this);
  }
```







# 应用裁剪Engine包

每个Flutter APK都包含Flutter引擎，占用包体积。由于嵌入式系统ROM固件是自己编译的，因此可以将引擎so库放到系统中，应用直接加载系统的so库，可以有效减小应用体积。

1. `flutter build apk --release --target-platform android-arm `打包apk，生成的so是arm架构的

2. 解压apk，删除`/lib/armeabi-v7a/libflutter.so`文件（打包好的Flutter引擎，arm架构约6.5M），删除`META-INF`下的签名文件

3. 重新打包签名，如果targetSDK为30的话，可能会报如下错误，需要使用`zipalign`对齐apk再签名

   ```shell
   Failure [-124: Failed parse during installPackageLI: Targeting R+ (version 30 and above) requires the resources.arsc of installed APKs to be stored uncompressed and aligned on a 4-byte boundary]
   ```

4. 安装apk，报错如下：由于应用中的Flutter引擎被裁剪，因此找不到so库。

   ```shell
   E/FlutterLoader: Flutter initialization failed.
       #...
        Caused by: java.lang.UnsatisfiedLinkError: dlopen failed: library "libflutter.so" not found
           at java.lang.Runtime.loadLibrary0(Runtime.java:1087)
           at java.lang.Runtime.loadLibrary0(Runtime.java:1008)
           at java.lang.System.loadLibrary(System.java:1664)
           at io.flutter.embedding.engine.FlutterJNI.loadLibrary(Unknown Source:13)
           at io.flutter.embedding.engine.h.c$a.a(Unknown Source:14)
           at io.flutter.embedding.engine.h.c$a.call(Unknown Source:0)
           at java.util.concurrent.FutureTask.run(FutureTask.java:266)
           at java.util.concurrent.ThreadPoolExecutor.runWorker(ThreadPoolExecutor.java:1167)
           at java.util.concurrent.ThreadPoolExecutor$Worker.run(ThreadPoolExecutor.java:641)
           at java.lang.Thread.run(Thread.java:923)
   ```

5. 将刚才删掉的`libflutter.so`文件push到`/system/lib`下，作为系统链接库

   > 也可以从`Flutter/flutter/bin/cache/artifacts/engine/`中获取缓存的引擎，注意so架构需要和目标平台匹配
   >
   > Android提供的`System.loadLibrary`是经过封装的，会从`ClassLoader`中获取`libraryPath`找到对应的so库，然后调用JNI的`nativeLoad`方法链接so库。
   >
   > 默认读取so库路径为`/vendor/lib`、`/system/lib`和`data/app/`下的app路径

6. 再次运行，报错如下：没有权限访问系统so库（Android高版本限制访问未知系统so库）

   ```shell
   Caused by: java.lang.UnsatisfiedLinkError: dlopen failed: library "/system/lib/libflutter.so" needed or dlopened by "/apex/com.android.art/lib/libnativeloader.so" is not accessible for the namespace "classloader-namespace"
           at java.lang.Runtime.loadLibrary0(Runtime.java:1087)
           at java.lang.Runtime.loadLibrary0(Runtime.java:1008)
           at java.lang.System.loadLibrary(System.java:1664)
           at io.flutter.embedding.engine.FlutterJNI.loadLibrary(Unknown Source:13)
           at io.flutter.embedding.engine.h.c$a.a(Unknown Source:14)
           at io.flutter.embedding.engine.h.c$a.call(Unknown Source:0)
           at java.util.concurrent.FutureTask.run(FutureTask.java:266)
           at java.util.concurrent.ThreadPoolExecutor.runWorker(ThreadPoolExecutor.java:1167)
           at java.util.concurrent.ThreadPoolExecutor$Worker.run(ThreadPoolExecutor.java:641)
           at java.lang.Thread.run(Thread.java:923)
   ```

7. 修改`/system/etc/public.libraries.txt`文件，添加`libflutter.so`，然后重启系统

8. 再次运行成功
